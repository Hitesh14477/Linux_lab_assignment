5. File Descriptor Observations

Observation:
Linux uses file descriptors to manage access to files, devices, pipes, and sockets. Standard input, standard output, and standard error are represented by file descriptors 0, 1, and 2 respectively.

6. Explanation of Linux File I/O

Linux manages file input and output through file descriptors. Every process receives standard input (0), standard output (1), and standard error (2). Commands can redirect these streams to files, making it easier to capture program output, log errors, and process data efficiently.

Summary

The investigation demonstrated how Linux tracks open files, redirects output and errors, and manages system resources. File descriptors provide a consistent interface for reading from and writing to files, making Linux I/O flexible and efficient.
