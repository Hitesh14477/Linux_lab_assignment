8. Comparison of Hard Link and Symbolic Link

Hard Link:
- Shares the same inode as the original file.
- Continues to access the data after the original filename is deleted.
- Cannot span different file systems.

Symbolic Link:
- Has its own inode.
- Stores the pathname of the target file.
- Becomes a broken link if the original file is deleted.
- Can point to files on different file systems.

9. Conclusions

The experiment demonstrates the key differences between hard links and symbolic links. Hard links directly reference the same file data and remain usable after deleting the original filename. Symbolic links only store the path to the original file, so they become invalid when the target file is removed.
